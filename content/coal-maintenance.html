Solving a Analytical Problem with Jupyter Notebook¶

Bernard Ade¶

A coal company in Western Australia requires some analytical work. They operate 5 enormous machines that stacks and reclaims mined coals 24-hours/ 365 days. They can't afford to let these machines run idle or else they come up short by millions in revenue.

After meeting with engineers and stakeholders, brainstorming, gathering requirements, a working plan to tackle the problem was conceived.

The challenge is to use the metric idle capacity to asesss which of the machines require maintenance. These machines require maintenance when within the previous month there was at least one 8-hour period when the average idle was over 10%.

The idle capacity metric is defined as follows:

$ {(Actual - Tonnes - Nominal - Capacity) \over Nominal - Capacity} $

We will break this process down into 3 steps.

  1. Inspect and clean data

  2. Compute metric

  3. Communicate our results.

I already created this assignment in Tableau. You can check it out here. Tableau is extremely fast and easy to conduct analysis of this nature. Working with data in Python is expressive, it helps to build the intuition behind data whereas in Tableau it all seems like magic.

Compute Metric¶

$ Idle-capacity = {(Actual - Tonnes - Nominal - Capacity) \over Nominal - Capacity} $

In [7]:
# calculate idle capacity metric
data['idle_capacity'] = (data['actual_tonnes'] - data['nominal_capacity'])/data['nominal_capacity']
In [8]:
# calculate moving averge with a 8-hour window
data['moving_average'] = data.idle_capacity.rolling(window=8,center=False).mean()
